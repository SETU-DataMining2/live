Permutation Feature Importance (PFI)

Outline

Permutation Feature Importance is a:

  • model-agnostic — it is not restricted to a certain model type, and
  • global — it only provides explanations for a full dataset, but not for individual samples.

Permutation Feature Importance was introduced in 2001 by Breiman et al. for Random Forest models and extended to a model-agnostic version in 2018 by Fischer et al.

Process

Permutation Feature Importance is applied as follows:

  1. Build a model to predict outputs and computes a performance measure for the dataset.

    • This will serve as baseline performance.
    • This model should be well preforming since feature importance is based on the model.

  2. Loop over each feature (potentially repeated multiple times):

    1. Permute the features value across the dataset, essentially messing up any connection between the label and this feature.

    2. Predict outputs and compute the performance measure based on the dataset with the permuted feature.

    3. Compare the performance measure with the baseline performance.

    4. Use the difference between baseline performance and permuted performance as an indicator of the importance of the feature.

    5. If both performances are similar, the permuted feature did not lead to a decrease in model performance, indicating that the model did not rely heavily on the feature, hence assigning low importance. On the other hand, if the performance of the data with the permuted feature is much worse than the baseline performance, this shows that the model highly depended on the feature to produce good scores.
