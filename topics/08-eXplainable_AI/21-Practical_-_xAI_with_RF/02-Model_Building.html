clf = RandomForestClassifier(oob_score=True, random_state=42, n_jobs=-1)

gs = GridSearchCV(clf, hyper_parm_space, cv=5, verbose=1, n_jobs=-1)
gs.fit(X_train, y_train)

Note the parameter, oob_score. This stands for Out-of-Bag score and you can treat this as a validation score for the model. It is calculated using the data samples not used during training of the trees. Recall, that RandomForest algorithm uses bootstrap sample of the data, which means that some samples are left out (the "out-of-bag" samples). The OOB score is calculated by using these left-out samples to evaluate the performance of the model. This is a useful feature of RandomForest as it allows us to get an estimate of the model's performance without needing to set aside a separate validation set.

Feature Correlation Model

We know from our PCA topic that the feature in this dataset are (highly) correlated so we either could do PCA (does a great job in dropping the dimension, but the resulting features are harder to understand) or we could do feature selection. We spoke before about the downside of highly correlated features on model training but it also affects explainability analysis. If two features are highly correlated, it can be difficult to determine which feature is more important for the model's predictions. This is because the model may use both features interchangeably, making it hard to attribute importance to one feature over the other.

So lets build a second model, where we use a subset of the features that are uncorrelated with each other ...

One way to select features that are uncorrelated is by performing hierarchical clustering on the Spearman rank-order correlations, picking a threshold, and keeping a single feature from each cluster (taken from sklearn documentation).

Using functions in plot_lib (imported as pl) we first plot a correlation heatmap the degree of correlation in our dataset

1
pl.plot_correlation_matrix(X,annot=False)

We can see in the heatmap above that we have many highly correlated features in our dataset. To select a set of features, which are uncorrelated, we plot the hierarchical clustering and manually pick a threshold by visual inspection of the dendrogram to group our features into clusters and choose a feature from each cluster to keep.

from scipy.stats import spearmanr
from scipy.cluster import hierarchy
from scipy.spatial.distance import squareform

# Compute Spearman correlation and make sure the matrix is symmetric
corr = spearmanr(X).correlation
corr = (corr + corr.T) / 2
np.fill_diagonal(corr, 1)

# Convert the correlation matrix to a distance matrix before 
# performing hierarchical clustering using Ward's linkage (see Bernard's notes from DM 1)
distance_matrix = 1 - np.abs(corr)
dist_linkage = hierarchy.ward(squareform(distance_matrix))

Finally plot using

1
pl.plot_dendrogram(dist_linkage, X.columns.to_list())

Looking at the dendrogram above, we choose a threshold of distance 1, for the grouping of features and randomly select the first feature in each cluster for our uncorrelated feature set.
